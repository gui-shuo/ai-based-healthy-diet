### 3. 💰 文本识别不消耗百度AI额度

**重要说明**：✅ **文本识别完全免费，不消耗百度AI额度！**

#### 工作原理

**文本识别流程**：
```
用户输入食物名称
    ↓
1. 数据库全文搜索（MySQL FULLTEXT）
    ↓
   找到？
    ↓
   是 → 返回数据库数据（置信度 95%）
    ↓
   否 → AI估算营养数据（置信度 70%）
    ↓
返回结果（不调用百度API）
```

**图片识别流程**：
```
用户上传图片
    ↓
调用百度AI图像识别API ⚠️ 消耗额度
    ↓
识别出食物名称列表
    ↓
对每个食物：
  1. 先查数据库
  2. 没有则AI估算
    ↓
返回结果
```

#### 代码证明

**后端代码**（`FoodRecognitionServiceV2.java`）：

```java
// 文本识别 - 不调用百度API
public FoodRecognitionResult recognizeByName(Long userId, String foodName) {
    // 1. 数据库搜索
    List<FoodNutrition> dbResults = foodNutritionRepository.fullTextSearch(foodName);
    
    if (!dbResults.isEmpty()) {
        // 数据库找到 - 直接返回
        return buildResult(dbResults);
    } else {
        // 数据库没找到 - AI估算（本地计算，不调用外部API）
        FoodItem item = estimateNutritionByAI(foodName);
        return buildResult(item);
    }
}

// 图片识别 - 调用百度API ⚠️
public FoodRecognitionResult recognizeByImage(Long userId, MultipartFile image) {
    // 调用百度AI识别图片
    JSONObject result = aipImageClassify.dishDetect(imageBytes, new HashMap<>());
    // ... 消耗百度AI额度
}
```

#### 额度使用建议

**免费额度**：1000次/天（百度AI）

**节省额度的方法**：
1. ✅ **优先使用文本识别**（完全免费）
   - 输入："苹果"、"鸡胸肉"、"燕麦" 等
   - 使用快捷输入按钮
   
2. ⚠️ **谨慎使用图片识别**（消耗额度）
   - 只在不知道食物名称时使用
   - 每次图片识别消耗 1 次额度

**数据来源标识**：
- `database`：数据库数据（最准确，免费）
- `estimated`：AI估算（较准确，免费）
- 图片识别后的数据库/估算（消耗1次额度后的免费查询）
