## 🎯 快速开始

### 新成员入门路径

**第1天**：了解项目背景
1. 阅读 [01-系统概述与目标](./01-系统概述与目标.md)
2. 了解项目愿景和核心价值

**第2天**：熟悉功能需求
1. 阅读 [02-功能需求详述](./02-功能需求详述.md)
2. 阅读 [02-功能需求详述-AI与后台](./02-功能需求详述-AI与后台.md)
3. 理解6大核心模块

**第3天**：掌握技术规范
1. 前端开发：阅读 [03-前端开发规范](./03-前端开发规范.md) + [08-UI设计规范](./08-UI设计规范.md)
2. 后端开发：阅读 [04-技术架构设计](./04-技术架构设计.md)
3. 数据库：阅读 [05-数据库设计](./05-数据库设计.md)
4. 技术选型：阅读 [09-技术选型确认](./09-技术选型确认.md)

**第4天**：熟悉接口规范
1. 阅读 [06-接口文档](./06-接口文档.md)
2. 了解API调用方式

**第5天**：制定工作计划
1. 阅读 [07-项目开发计划](./07-项目开发计划.md)
2. 明确自己的任务和时间节点

## 📝 文档贡献指南

### 如何更新文档

1. **克隆仓库**
   ```bash
   git clone [仓库地址]
   cd ai-based-healthy-diet/docs
   ```

2. **创建分支**
   ```bash
   git checkout -b docs/update-xxx
   ```

3. **编辑文档**
   - 使用Markdown格式
   - 遵循现有文档结构
   - 添加必要的图表和示例

4. **提交更新**
   ```bash
   git add .
   git commit -m "docs: 更新XXX文档"
   git push origin docs/update-xxx
   ```

5. **创建Pull Request**
   - 描述更新内容
   - 标注关联的Issue（如有）
   - 等待Review

### 文档规范

- **标题层级**: 使用1-4级标题，不超过4级
- **代码块**: 必须指定语言类型（如 ```javascript）
- **表格**: 确保对齐且易读
- **图表**: 优先使用Mermaid语法
- **链接**: 使用相对路径链接其他文档
- **示例**: 提供完整可运行的代码示例
