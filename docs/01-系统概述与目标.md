## 1. 项目背景

### 1.1 行业现状
据《中国居民营养与慢性病状况报告》显示：
- **73%** 的都市人群存在饮食结构不合理问题
- **专业营养师服务覆盖率** 不足2%
- **慢性病患病率** 持续上升，与不健康饮食习惯密切相关
- **健康管理需求** 呈现爆发式增长，但专业服务供给严重不足

### 1.2 用户痛点分析
| 痛点类别 | 具体表现 | 影响程度 |
|---------|---------|---------|
| 知识匮乏 | 不知道如何科学搭配三餐 | ⭐⭐⭐⭐⭐ |
| 时间成本 | 无法持续跟踪营养摄入情况 | ⭐⭐⭐⭐ |
| 经济成本 | 专业营养师服务价格昂贵（300-800元/次） | ⭐⭐⭐⭐⭐ |
| 个性化难 | 通用饮食建议无法适配个人体质 | ⭐⭐⭐⭐ |
| 坚持困难 | 缺乏激励机制和习惯养成支持 | ⭐⭐⭐⭐⭐ |

### 1.3 市场机遇
- **AI技术成熟**：大语言模型在营养咨询领域的应用日趋成熟
- **政策支持**：《健康中国2030规划纲要》明确提出营养健康产业发展方向
- **消费升级**：Z世代和千禧一代健康消费意愿强烈
- **技术普及**：移动互联网和智能设备普及率超过90%

## 2. 核心价值主张

### 2.1 产品定位
**AI健康饮食规划助手** = 专业营养师 × AI智能化 × 会员成长体系

一个为都市人群提供7×24小时专业饮食规划服务的智能健康管理平台，通过AI技术将专业营养学知识转化为可执行的日常饮食计划。

### 2.2 核心价值
1. **降低门槛**：将专业营养咨询服务成本降低95%以上
2. **提升效率**：3分钟内生成个性化饮食方案，传统方式需1-2小时
3. **动态调整**：基于用户反馈和身体数据变化实时优化方案
4. **习惯养成**：通过会员成长体系和智能提醒培养健康习惯
5. **数据驱动**：建立个人健康档案，长期跟踪营养摄入趋势

### 2.3 竞争优势
| 竞争维度 | 传统营养师 | 通用健康App | 本系统 |
|---------|-----------|------------|--------|
| 成本 | 300-800元/次 | 免费（缺乏深度） | 月费50-200元 |
| 响应速度 | 预约需1-3天 | 即时（非专业） | 即时（专业AI） |
| 个性化程度 | 高 | 低 | 高（AI学习） |
| 持续跟踪 | 有限（成本高） | 基础 | 全面（自动化） |
| 数据分析 | 人工分析 | 简单统计 | AI深度分析 |

### 4.3 权限控制实现
```javascript
// 前端路由守卫示例
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  const requiredRole = to.meta.role
  
  if (to.meta.requiresAuth && !authStore.token) {
    // 未登录，跳转登录页
    next('/login')
  } else if (requiredRole && !authStore.hasRole(requiredRole)) {
    // 权限不足，提示错误
    ElMessage.error('权限不足')
    next(false)
  } else {
    next()
  }
})
```

## 5. 项目范围

### 5.1 功能范围
**包含**：
- ✅ 6大核心功能模块（首页、登录注册、个人中心、会员中心、AI功能、后台管理）
- ✅ AI饮食规划生成（基于通义千问API）
- ✅ 健康档案管理
- ✅ 会员成长体系
- ✅ 数据可视化分析
- ✅ 实时消息推送

**不包含**：
- ❌ 在线支付功能（第二期实现）
- ❌ 社区论坛功能（第二期实现）
- ❌ 食材电商功能（第三期实现）
- ❌ 移动端原生APP（当前仅H5响应式）
- ❌ 线下营养师预约（未来规划）

### 5.2 技术边界
**前端技术栈**：
- 框架：Vue 3.3+
- 状态管理：Pinia
- 路由：Vue Router 4
- UI组件库：Element Plus
- 样式：TailwindCSS + SCSS
- 图表：ECharts
- 工具库：Axios、Day.js、Lodash

**后端技术栈**：
- 框架：Spring Boot 3.2
- AI框架：LangChain4j
- 数据库：MySQL 8.0
- 缓存：Redis 6.0
- 消息队列：RabbitMQ（用于异步AI请求）
- AI服务：阿里云通义千问API

### 5.3 环境要求
| 环境类型 | 配置要求 |
|---------|---------|
| 开发环境 | Node.js 18+, JDK 17+, MySQL 8.0, Redis 6.0 |
| 测试环境 | 1核2G ECS × 1, RDS MySQL, Redis |
| 生产环境 | 2核4G ECS × 2, RDS MySQL主从, Redis集群 |
| 浏览器支持 | Chrome 100+, Safari 15+, Edge 100+ |
