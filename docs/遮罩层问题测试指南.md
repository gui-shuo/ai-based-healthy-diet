**测试日期**: 2025年12月5日

## 测试环境

- **前端服务器**: http://localhost:3000

## 调试技巧

### 使用 Chrome DevTools

1. 打开 Chrome DevTools（F12）
2. 切换到 "Console" 标签
3. 观察控制台输出，查找以下日志：
   - `FoodRecognitionView 组件卸载，开始清理...`
   - `清理 MessageBox`
   - `清理遮罩层`
   - `路由跳转: /food-recognition -> /ai-chat`

### 检查 DOM 元素

1. 打开 Chrome DevTools（F12）
2. 切换到 "Elements" 标签
3. 在页面上右键点击灰色遮罩层（如果有）
4. 选择 "检查元素"
5. 查看元素的类名和属性：
   - `.el-overlay` - Element Plus 遮罩层
   - `.el-loading-mask` - 加载遮罩层
   - `.el-message-box__wrapper` - MessageBox 包装器
   - `.el-dialog__wrapper` - Dialog 包装器

### 手动清理遮罩层

如果测试过程中出现遮罩层残留，可以在控制台执行以下命令手动清理：

```javascript
// 清理所有遮罩层
document.querySelectorAll('.el-overlay').forEach(el => el.remove())

// 清理所有 MessageBox
document.querySelectorAll('.el-message-box__wrapper').forEach(el => el.remove())

// 清理所有 Dialog
document.querySelectorAll('.el-dialog__wrapper').forEach(el => el.remove())
```

## 常见问题

### Q1: 遮罩层仍然存在怎么办？

**A**: 
1. 检查控制台是否有错误日志
2. 检查是否有其他弹窗组件（如 Notification、Message）
3. 尝试刷新页面
4. 检查 `App.vue` 中的清理逻辑是否正确执行

### Q2: 页面切换后出现白屏怎么办？

**A**: 
1. 检查控制台是否有 JavaScript 错误
2. 检查路由配置是否正确
3. 检查组件是否正确加载

### Q3: 如何验证清理逻辑是否执行？

**A**: 
1. 打开控制台，查看日志输出
2. 在 `onBeforeUnmount` 中添加 `console.log` 输出
3. 使用 Chrome DevTools 的 "Performance" 标签记录页面切换过程
