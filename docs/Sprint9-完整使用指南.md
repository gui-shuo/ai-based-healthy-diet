# Sprint 9 - AI饮食计划生成功能完整使用指南

## 🎯 功能概述

本次Sprint实现了一个**细化的AI饮食计划生成功能**，用户可以通过步骤化选择来定制个性化的饮食计划。

### 核心特性

1. **步骤化选择** - 3步完成配置
2. **目标选择** - 减脂/增肌/健康维持
3. **天数选择** - 7天/14天/30天
4. **可选功能** - 营养数据/采购清单/烹饪提示
5. **实时预览** - 显示当前配置
6. **一键生成** - 智能生成完整计划

## 🚀 使用步骤

### 步骤1: 登录系统

1. 访问：http://localhost:3001
2. 点击右上角"登录"按钮
3. 输入账号密码：
   - 用户名：`admin`
   - 密码：`Admin123456`
4. 点击"登录"

### 步骤2: 进入AI饮食计划页面

有**3种方式**进入：

#### 方式1: 通过用户菜单（推荐）
1. 点击右上角用户名下拉菜单
2. 选择"**AI饮食计划**"

#### 方式2: 通过首页功能卡片
1. 在首页向下滚动
2. 找到"核心功能"区域
3. 点击"**AI饮食计划**"卡片

#### 方式3: 直接访问URL
```
http://localhost:3001/diet-plan
```

### 步骤3: 配置饮食计划

进入页面后，您会看到3个配置步骤：

#### 🎯 步骤1: 选择目标

选择您的健康目标（三选一）：

| 目标 | 说明 | 适合人群 |
|------|------|----------|
| **减脂塑形** 🔥 | 控制热量，减少脂肪 | 需要减重的用户 |
| **增肌强壮** 💪 | 高蛋白，增加肌肉 | 健身增肌用户 |
| **健康维持** 🌟 | 均衡营养，保持健康 | 一般健康用户 |

**操作**：点击对应的单选按钮

#### 📅 步骤2: 选择天数

选择计划的天数（三选一）：

| 天数 | 说明 | 生成时间 |
|------|------|----------|
| **7天计划** | 一周计划，快速体验 | ~30-60秒 |
| **14天计划** | 两周计划，短期目标 | ~60-90秒 |
| **30天计划** | 一月计划，长期规划 | ~90-120秒 |

**操作**：点击对应的单选按钮

#### ⚙️ 步骤3: 选择生成选项

勾选您需要的功能（可多选）：

| 选项 | 说明 | 效果 |
|------|------|------|
| ✅ **详细营养数据** | 每个食材的热量、蛋白质、碳水、脂肪 | 精确的营养管理 |
| ✅ **生成采购清单** | 按类别汇总所有食材 | 方便一次性采购 |
| ✅ **包含烹饪提示** | 每餐的简单制作方法 | 易于操作实施 |

**默认**：全部勾选

**操作**：点击复选框勾选/取消

### 步骤4: 查看配置预览

在配置区域下方，会实时显示当前配置：


**确认配置无误后，点击"立即生成饮食计划"按钮**

## 🔍 功能详解

### 1. 营养数据开关

**开启时**：
```markdown
- **燕麦**：50g - 183.5kcal，蛋白质7.5g，碳水33.45g，脂肪3.35g
- **小计**：365 kcal
```

**关闭时**：
```markdown
- **燕麦**：50g
- （无小计）
```

### 2. 采购清单开关

**开启时**：
- 计划末尾会生成完整的采购清单
- 按类别分组（蔬菜、水果、肉类等）
- 汇总所有天数的食材

**关闭时**：
- 不生成采购清单
- 计划末尾直接结束

### 3. 烹饪提示开关

**开启时**：
```markdown
### 早餐 (7:00-8:00)
- **燕麦**：50g
- **烹饪提示**：燕麦提前浸泡，早上加牛奶煮沸即可
```

**关闭时**：
```markdown
### 早餐 (7:00-8:00)
- **燕麦**：50g
（无烹饪提示）
```

## ⚠️ 常见问题

### Q1: 页面找不到"AI饮食计划"入口？

**解决方案**：
1. 确保已登录
2. 刷新页面：`Ctrl + Shift + R`（强制刷新）
3. 检查浏览器控制台是否有错误
4. 确认前端服务正在运行

### Q2: 点击"立即生成"后没有反应？

**可能原因**：
1. 后端服务未启动
2. 数据库连接失败
3. API Key未配置

**解决方案**：
```bash
# 1. 检查后端是否运行
netstat -ano | findstr :8080

# 2. 检查后端日志
# 查看控制台输出

# 3. 检查环境变量
echo %TONGYI_API_KEY%
```

### Q3: 生成时间太长？

**正常时间**：
- 7天：30-60秒
- 14天：60-90秒
- 30天：90-120秒

**如果超过2分钟**：
1. 检查网络连接
2. 检查AI服务是否正常
3. 刷新页面重试

### Q4: 生成的内容不完整？

**检查项**：
1. 是否勾选了对应的选项？
2. 向下滚动查看完整内容
3. 采购清单在最底部，需要滚动到末尾

### Q5: 采购清单没有显示？

**解决方案**：
1. 确认勾选了"生成采购清单"选项
2. 向下滚动到内容最底部
3. 查找分隔线 `---` 之后的内容
